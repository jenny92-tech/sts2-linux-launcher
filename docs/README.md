# docs/

Development notes, debug logs, and post-mortem records.

This directory exists separately from the project root README so the
main README stays a tight project description. Anything that would
distract a first-time reader — patch experiment notes, device-specific
test results, regression debugging — belongs here.

Each note is its own markdown file with a date prefix:

```
docs/
├── 2026-06-11-globalization-invariant-cascade.md
├── 2026-06-11-hostfxr-tpa-missing-deps.md
└── ...
```
